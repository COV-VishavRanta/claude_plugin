# Shared Agent Conventions

## Purpose

This skill defines cross-cutting conventions shared by all architecture agents in this project. It is read as item 0 in every agent's parallel input batch and applies universally to all output files written under `docs/architecture/`.

---

## Confidence Annotation System

Use these three labels to annotate every decision, table cell, diagram node, and inferred value in your output:

| Label | Meaning |
|---|---|
| ✅ `DECIDED` | Explicitly stated in input documents; no ambiguity |
| ⚠️ `INFERRED` | Not stated, but reasonably derived from inputs — **always state the reason in parentheses** |
| ❓ `TBD` | Genuinely unknown; must surface as a bullet in the file's Open Questions section |

---

## Generation Header Format

Every output file must begin with a generation header on the first line after the file's `#` title (or as the very first line if there is no title):

```
> Generated by [agent-name] agent | Date: YYYY-MM-DD | Version: vX.X
```

- Replace `[agent-name]` with the agent's canonical name (e.g., `backend-architecture`, `db-architecture`, `cloud-architecture`, `frontend-architecture`, `project-blueprint`).
- Set version to `v1.0` on first creation.
- Increment by one minor version on each update (`v1.0` → `v2.0` → `v3.0`).

---

## Mode Detection Protocol

Before writing any output file, check the agent's target directory to determine the correct write action:

- **MKDIR:** Target directory does not exist → create the directory first before writing any files into it
- **CREATE:** File does not exist → use `create_file`; set version `v1.0`
- **UPDATE:** File already exists → read it first, then use `multi_replace_string_in_file`; increment the version

**Write all output files in ONE parallel tool call. Never call `create_file` on an existing file.**

Each agent specifies its own output directory. Apply this protocol to that directory.

---

## "Use Defaults" Fallback Behavior

If the user says **"use defaults"** or **"decide for me"**, infer all open decisions from the available inputs, annotate each inferred decision with ⚠️ `INFERRED` (state the reason), and proceed without asking further questions.

---

## Open Questions Discipline

- Every output file must include an **Open Questions** section with a maximum of **6 bullets**.
- Every ❓ `TBD` decision must appear as a bullet in that section — never carry unresolved decisions forward silently.
- Surface gaps found in missing or incomplete input documents as Open Questions rather than making silent assumptions.
